    Pipeline

    How The Forecast Was Built

    01

    Aggregate Container Rows

    Convert container-level reefer records into one terminal-level hourly target with load, container counts, temperature summaries, stack-tier proxies, and customer mix.

    02

    Build Causal Features

    Create lagged load signals, rolling statistics, calendar features, and reefer-state proxies using past information only.

    03

    Train Recursive Models

    Train a stable baseline recursive model and a recursive XGBoost model, then forecast the public target block hour by hour.

    04

    Blend For Final Submission

    Combine the baseline and XGBoost forecasts with a two-regime rule so lower-load hours lean more on the baseline and peak-like hours lean more on XGBoost.

    Final Choice

    Why the v2 blend won

    The baseline model remained stable under recursive forecasting, while the recursive XGBoost model improved peak behavior. The final two-regime blend captured both strengths more effectively than a single fixed blend weight.

    Leakage Handling

    Why the core setup is still causal

    The two main recursive models are trained before the target block and forecast forward. The final blend weights were tuned on the visible public period, but the base models themselves are recursive and history-driven.

    LLM Usage

    Where AI helped

    LLMs were used for feature brainstorming, code generation, experiment design, and documentation support. Final modeling choices were validated with time-based testing.
